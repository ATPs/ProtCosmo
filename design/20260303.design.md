# ProtCosmo Design (2026-03-03)

## 1. Goal
ProtCosmo is a CLI pipeline that:
1. Runs CometPlus search for one or more mass files.
2. Reads Comet PIN candidates.
3. Applies static linear scoring using Percolator weight files.
4. Selects one winner PSM per spectrum.
5. Estimates q-value and PEP using score lookup against reference Percolator outputs.
6. Writes all-winner and novel-focused report files.

`src/protcosmo/protcosmo.py` is the entrypoint, and reusable helpers are in `src/protcosmo/utils`.

## 2. Program Flow (What happens first/next)

### Step 1: Parse CLI
1. Parse ProtCosmo options using `argparse`.
2. Collect unknown `--<param>` arguments as passthrough CometPlus parameter overrides.
3. `--help` shows concise help.
4. `--help-full` shows concise + detailed behavior notes.

### Step 2: Build normalized run configuration
1. If `--input_tsv` is provided, read it (ignore lines starting with `#`).
2. Otherwise, build config from CLI values.
3. Apply CLI overrides on top of TSV values.
4. Expand each row into per-mass-file runs.
5. Mapping rule for list-like fields (`params`, `database`, `init-weights`, `percolator-psms`, `percolator-peptides`):
   - 1 value => broadcast to all mass files in that row.
   - N values => 1:1 mapped by order.
   - Other counts => error.
6. If total mass-file count > 1, scan-related args are ignored with warning:
   - `--scan`
   - `--scan_numbers`
   - `--first-scan`
   - `--last-scan`

### Step 3: Run CometPlus per mass file
For each normalized run:
1. Build command with required arguments.
2. Force `--output_percolatorfile 1`.
3. Force `--max_duplicate_proteins -1`.
4. Set per-run output base (`--name`) inside `<output-dir>/comet_outputs/run_xxxx/`.
5. Execute command and capture stdout/stderr logs.
6. Detect generated PIN file (`.pin`, `.pin.gz`, `.pin.parquet`, `.pin.parquet.gz`).

### Step 4: Load PIN and static models
1. Read PIN table (text/gzip/parquet supported).
2. Parse Percolator weights from `--init-weights`.
3. Use numeric rows **1, 3, 5**.
4. For each model row: `score = w^T x + b`.
5. Final candidate score = mean of the 3 model scores.

### Step 5: Select winner per spectrum
1. Derive spectrum key from `SpecId` without rank suffix.
2. Sort candidates by:
   - higher `final_score`
   - non-novel before novel on ties
   - lower rank index on remaining ties
3. Keep one winner per spectrum.

### Step 6: Estimate q-value/PEP for winners
1. Load per-run reference table from `--percolator-psms`.
2. For each winner score, choose reference row with largest score `<=` winner score.
3. Copy that row's q-value and PEP as estimated values.
4. Fallback rule when no smaller score exists: `q-value=1`, `PEP=1`.

### Step 7: Build novel summaries
1. Mark novel-only winners by protein ids named `COMETPLUS_NOVEL_*` (decoy prefix `DECOY_` is stripped for this check).
2. Create novel PSM subset.
3. Derive peptide forms:
   - modified-level peptide: strip PIN flanks only
   - unmodified peptide: strip brackets/mod markers and collapse to amino-acid letters
4. Load `--percolator-peptides` lookup and estimate peptide-level q/PEP by same closest-smaller rule.
5. Aggregate summaries by:
   - modified peptide
   - unmodified peptide
   - novel protein id

### Step 8: Write output files
Write all report files, warning log, and metadata JSON to `--output-dir`.

## 3. Parameter Definitions

### Core run inputs
- `--cometplus`: path to CometPlus executable. Default: `cometplus`.
- `--mass-file`: one mass file or comma-joined list.
- `--params`: one params file or comma-joined list.
- `--database`: one database file or comma-joined list.
- `--init-weights`: one weights file or comma-joined list.
- `--percolator-psms`: one reference target PSM file or comma-joined list (tsv/parquet).
- `--percolator-peptides`: one reference target peptide file or comma-joined list (tsv/parquet).
- `--output-dir`: output directory for reports and intermediate run logs.

### Novel inputs
- `--novel_protein`: novel protein FASTA for CometPlus novel mode.
- `--novel_peptide`: novel peptide input file (FASTA or tokenized text, as supported by CometPlus).

### Scan filters (single-file mode only)
- `--scan`: scan file.
- `--scan_numbers`: comma-separated scan numbers.
- `--first-scan`: start scan.
- `--last-scan`: end scan.

### Other
- `--thread`: pass thread override to CometPlus.
- `--input_tsv`: optional TSV defining one or many runs.
- `--help`: concise help.
- `--help-full`: detailed behavior help.

### Passthrough behavior
Unknown `--<param>` arguments are passed directly to CometPlus, enabling core Comet parameter overrides without hardcoding each key in ProtCosmo.

## 4. Input TSV format

Header-based TSV; lines starting with `#` are ignored.

Supported columns:
- `mass-file`
- `params`
- `database`
- `init-weights`
- `percolator-psms`
- `percolator-peptides`

Notes:
1. Column aliases with underscore are also accepted (for example `mass_file`).
2. CLI values override TSV values field-by-field.

## 5. Scoring and QC estimation details

### Static score
For each PIN candidate with feature vector `x`:

`score_k = w_k^T x + b_k` for `k in {1,3,5}` numeric rows from weights file

`final_score = mean(score_1, score_3, score_5)`

### q-value/PEP estimation
Given winner score `s` and reference rows sorted by `score`:
1. Find row with maximum `score_ref` such that `score_ref <= s`.
2. Use that row's q-value and PEP.
3. If no such row exists, use fallback `(1,1)`.

Important caveat:
- These are **estimated** values by lookup mapping, not recalculated Percolator confidence values.
- Real values are expected to be smaller than estimates.

## 6. Output Files and Meanings

ProtCosmo writes these files under `--output-dir`:

1. `protcosmo.novel_psms.tsv`
   - Subset where winner is novel-only.
2. `protcosmo.novel_peptides.modified.tsv`
   - Aggregated by modified peptide form.
3. `protcosmo.warnings.log`
   - Runtime warnings (for example multi-file scan ignore, fallback assignments).
4. `protcosmo.run_metadata.json`
   - Full execution metadata:
     - start/end times
     - commands per run
     - input/output paths
     - counts and warnings

Intermediate Comet outputs/logs are under:
- `comet_outputs/run_xxxx/`

## 7. Usage Examples

### Example A: single file with scan filters
```bash
PYTHONPATH=src /data/p/anaconda3/bin/python -m protcosmo.protcosmo \
  --cometplus /data/p/comet/Comet/ProtCosmo/CometPlus/cometplus \
  --mass-file /path/to/input.mzMLb \
  --params /path/to/comet.params \
  --database /path/to/database.idx \
  --scan_numbers 1001,1002,1003 \
  --first-scan 1 --last-scan 2000 \
  --init-weights /path/to/model.weights \
  --percolator-psms /path/to/target.psms.parquet \
  --percolator-peptides /path/to/target.peptides.parquet \
  --output-dir /path/to/output
```

### Example B: multiple mass files with broadcast settings
```bash
PYTHONPATH=src /data/p/anaconda3/bin/python -m protcosmo.protcosmo \
  --mass-file /path/a.mzMLb,/path/b.mzMLb \
  --params /path/comet.params \
  --database /path/database.idx \
  --init-weights /path/model.weights \
  --percolator-psms /path/target.psms.parquet \

  --output-dir /path/output
```

## 9. Limitations / Notes
1. Estimated q/PEP are lookup-based approximations.
2. If novel+scan settings are too restrictive, CometPlus can retain 0 spectra; the run fails with no-search warning.
3. Novel summaries can legitimately be empty when no novel-only winners are observed.
4. Scan filters are intentionally ignored when more than one mass file is provided.
